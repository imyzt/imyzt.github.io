Git 原理详解及实用指南 - HEAD，Branch

2020-02-17 1 min read # git

git branch、HEAD都是围绕着commit走的；

HEAD：永远指向当前commit，每个仓库只有一个HEAD，每次提交之后都会向前移动到最新的commit。远程分支HEAD永远指向默认分支master。

Branch：一类引用，HEAD除了直接指向commit之外，也可以通过指向branch间接指向commit。当HEAD指向branch时，commit发生时，HEAD会带着branch一起移动。如下图（HEAD携带feature1一起移动。）

master：master是Git中默认的branch，它与其他branch的区别在于：

  1. 新建仓库中第一个commit会被master自动指向。
  2. 在git clone时，会自动checkout到master。

branch的基本操作。

  1. 创建： git branch 名称，或 git checkout -b 名称
  2. 切换：git checkout 名称
  3. 删除：git branch -d/-D 名称，-D强制删除
      • master：master是Git中默认的branch，它与其他branch的区别在于：
      • branch的基本操作。
